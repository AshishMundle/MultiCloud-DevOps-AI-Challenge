## 💡 Key Learnings from Session 30

> **1.** Docker container is an isolated process with its own filesystem network and PID namespace
> **2.** Image is read-only template Container is a running instance of that image
> **3.** Docker Hub has 300000 plus public images — always use official images for security
> **4.** Containers share the host OS kernel making them much lighter and faster than VMs
> **5.** Docker permanently solves the it works on my machine problem across all environments
