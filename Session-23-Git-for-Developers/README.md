## 💡 Key Learnings from Session 23

> **1.** Never commit directly to main or master branch — always use feature branches
> **2.** Pull Request is the formal code review process before merging to main branch
> **3.** git rebase creates cleaner linear history git merge preserves full branch history
> **4.** git stash saves work in progress without committing — use during hotfix situations
> **5.** Branch naming convention: feat/ for features fix/ for bugs hotfix/ for production
