## 💡 Key Learnings from Session 25

> **1.** Maven is the build tool for Java projects — pom.xml defines all dependencies and plugins
> **2.** Declarative pipeline is recommended for most teams as it is easier to maintain
> **3.** Jenkinsfile in root of repo means Pipeline as Code — fully version controlled
> **4.** stages then stage then steps is the standard structure of every Declarative Jenkinsfile
> **5.** post always success failure blocks handle build status notifications and cleanup
