> **1.** terraform plan shows exactly what will change before applying — never skip this step
> **2.** State file tracks real infrastructure state — never delete or manually edit the state file
> **3.** Remote state in S3 plus DynamoDB lock is safe for team collaboration on shared infra
> **4.** Variables make your Terraform code reusable across dev staging and production
> **5.** terraform destroy removes all resources — use with extreme caution in production environments
