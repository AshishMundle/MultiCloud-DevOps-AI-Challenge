## 💡 Key Learnings from Session 32

> **1.** Every Dockerfile instruction creates a new layer — order of instructions matters for caching
> **2.** Put frequently changing instructions like COPY code at bottom for maximum cache reuse
> **3.** Multi-stage builds reduce final image size by 60 to 80 percent — use for production
> **4.** .dockerignore prevents copying node_modules and .git folder into the image
> **5.** docker inspect image_name shows full metadata including all layers and their sizes
