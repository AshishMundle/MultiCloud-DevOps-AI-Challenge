## 💡 Key Learnings from Session 17

> **1.** RDS is managed database — AWS handles backups patches failover automatically
> **2.** Multi-AZ RDS has standby in another AZ with automatic failover in 60 to 120 seconds
> **3.** Read Replicas reduce load on primary database — use them for reporting queries only
> **4.** Point-in-time recovery lets you restore to any second in the last 35 days
> **5.** Never connect directly to database from internet — always use EC2 as proxy or bastion host
