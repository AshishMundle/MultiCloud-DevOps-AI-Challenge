## 💡 Key Learnings from Session 43

> **1.** set -e at top of every script exits immediately on any error — essential for production safety
> **2.** dollar-question returns exit code: 0 means success non-zero means failure — always check it
> **3.** Shebang #!/bin/bash must be first line to tell the OS which interpreter to use
> **4.** Always double-quote variables as dollar-curly-brace-variable to prevent word splitting bugs
> **5.** Bash scripts stored in Git means version-controlled automation — the DevOps gold standard
