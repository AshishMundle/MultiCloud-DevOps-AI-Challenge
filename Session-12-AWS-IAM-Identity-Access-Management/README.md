## 💡 Key Learnings from Session 12

> **1.** Root account should never be used for daily work — create IAM admin user instead
> **2.** Principle of Least Privilege means give only minimum permissions absolutely needed
> **3.** IAM Roles are for services like EC2 and Lambda IAM Users are for human access
> **4.** Policy is a JSON document that defines what actions are allowed or explicitly denied
> **5.** MFA on root account is the single most important security step in all of AWS
