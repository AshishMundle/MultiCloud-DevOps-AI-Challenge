## 💡 Key Learnings from Session 51

> **1.** Microservices means each service can be deployed scaled and updated completely independently
> **2.** Each service in its own Kubernetes namespace for clean isolation and independent deployments
> **3.** Resource limits on pods prevent one bad service from consuming all cluster resources
> **4.** Liveness and readiness probes ensure traffic only routes to pods that are fully healthy
> **5.** Netflix Amazon and Google all run microservices on Kubernetes — you are doing what they do
