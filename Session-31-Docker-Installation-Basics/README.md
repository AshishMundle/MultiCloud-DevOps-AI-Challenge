## 💡 Key Learnings from Session 31

> **1.** docker run -d -p 80:80 nginx runs nginx in background and exposes port 80 to host
> **2.** docker exec -it container bash enters running container — most used daily command
> **3.** Volumes persist data even after container is deleted — always use for databases
> **4.** docker logs container_name checks container output just like tail -f in Linux
> **5.** docker ps -a shows all containers including stopped ones — clean up regularly
