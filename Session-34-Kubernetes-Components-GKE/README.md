## 💡 Key Learnings from Session 34

> **1.** Pod contains one or more containers sharing the same network namespace and storage
> **2.** Deployment manages ReplicaSet which manages Pods — three levels of abstraction
> **3.** Service provides stable endpoint for accessing Pods since Pods are ephemeral
> **4.** HPA scales pods automatically based on CPU or memory — set target at 70 percent CPU
> **5.** GKE is managed Kubernetes on Google Cloud — no master node management required
